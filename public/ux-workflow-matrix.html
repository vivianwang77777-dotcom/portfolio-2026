AI 赋能的 UX 工作流

核心把从发现问题到交付表达的全过程串起来，形成一套可复用的思维框架。

PHASE 01

建立业务背景，明确任务

PHASE 02

收敛问题，转化为验证假设

PHASE 03

产出架构、逻辑与视觉细化

PHASE 04

对比验证，得到可信结论

探索/验证新方向 需求拷问器与系统架构师

情况一：0-1 项目 (MVP 定义)

操作策略与核心步骤

“此时由于没有线上数据，重点在于“被动回答”。利用AI对你进行追问，逼出你自己都没想清楚的功能边界和成功标准。”

步骤 i. 需求澄清

告知AI你想做的产品雏形，让其扮演“需求拷问器”。通过AI的追问，明确目标用户、核心场景和功能边界。

步骤 ii. 定义 MVP最小可行性产品和完整需求文档

让AI作为顶级架构师，基于追问结果定义产品的MVP版本并输出严谨可开发的PRD。

生产级 PROMPT 代码库

prompt_logic_1.md 
一、你现在是需求拷问器，不是执行者。我要做的产品是：[一句话描述]
请开始追问我，直到所有关键问题清晰：-目标用户是谁 -核心使用场景 -功能边界(有什么、没什么) -成功标准。追问完毕后，输出一份需求总结。
prompt_logic_2.md 
二、你是一名「顶级产品架构师+UX专家+AI产品设计师+技术负责人」。你的任务是基于我提供的回答，定义这个产品的MVP版本，并生成一份完整、专业、结构严谨、可直接用于开发的PRD。
修复/提升型项目 数据诊断器与逻辑补齐器

情况二：基于线上优化 (根因定位)

操作策略与核心步骤

“识别 UI “症状” (Symptoms)，并将其抽象为设计挑战。”

步骤 i. 识别 UI 症状

上传截图，让AI从任务、场景、决策成本出发拆解体验问题。

步骤 ii. 驱动逻辑链

建立“业务背景=业务问题+市场趋势+机会判断”的推导逻辑。

生产级 PROMPT 代码库

prompt_logic_1.md 
你是一位资深 UX/产品体验专家，请不要只做表层视觉评价，而是必须从「用户真实任务+使用场景+交互优先级+决策成本」出发，系统性分析一个界面的体验问题。拆解这个页面为什么这样设计、哪里有问题、问题为什么严重、应该如何改。
prompt_logic_2.md 
你是一位资深 UX/产品体验专家，具备业务分析能力。请根据我提供的背景，推导清晰、可量化的业务目标，并拆解对应的体验目标（体验目标=设计目标+衡量指标）。
文档到方案精准对齐 逻辑提取器与方案发散器

情况三：PRD 承接项目 (架构转化)

操作策略与核心步骤

“将冗长的文字转化为可设计的结构，体现设计的决策深度。”

步骤 i. 系统性提取

从 PRD 提取功能点、用户路径，直接产出可执行的 DRD。

步骤 ii. 多路径探索

围绕目标发散 3 套思路完全不同、互不重叠的交互框架方案。

生产级 PROMPT 代码库

prompt_logic_1.md 
你是一名资深UX设计师。你的任务是从PRD中系统提取功能点、用户路径与线框图，并输出可直接执行的DRD；同时要求围绕产品目标发散提出三个思路完全不同、互不重叠的交互框架方案。每个DRD需以完整的产品用户体验设计视角进行呈现。
成熟系统新增功能 系统边界审计员

情况四：扩展型项目 (Expansion)

操作策略与核心步骤

“验证新功能是否会被发现、是否会干扰原有的核心路径与成熟系统的底层逻辑。”

步骤 i. 识别功能边界

评估入口优先级与展示层级，分析竞品同类场景插入方式。

步骤 ii. 建立验证指标

制定针对新功能的曝光率、激活率、复用率等全链路验证方案。

生产级 PROMPT 代码库

prompt_logic_1.md 
利用AI评估新功能入口在现有界面中的优先级，分析竞品如何插入类似功能。制定新功能的验证方案(曝光、激活、完成、复用率)，并审查新功能对现有UI是否造成视觉冲突与链路干扰。
专项微调提升转化 专项诊断专家

情况五：局部改动项目 (Local Change)

操作策略与核心步骤

“针对按钮文案、交互手势或视觉体系进行小范围优化，用确定性的逻辑撬动局部业务指标。”

步骤 i. 视觉基因提取

即使是局部改动，也需通过AI提取原有系统的 Design Tokens (主色、圆角、间距)，确保局部改动与整体一致。

步骤 ii. 微观验证计划

利用AI制定 A/B 测试或热力图分析计划，精准验证改动后的点击率或错误操作率变化。

生产级 PROMPT 代码库

prompt_logic_1.md 
通过AI提取原有系统的 Design Tokens (主色、圆角、间距)，确保局部改动与整体一致，并制定A/B测试或热力图分析计划，验证改动后的点击率或错误操作率变化。
Framework & Logic 信息架构师 & 竞品调研专家

结构与调研

操作策略与核心步骤

“改变主观判断，利用 AI 统计“设计变量”频率得出概率级有效结论。”

步骤 i. 结构设计 (IA)

根据需求自动输出页面清单、跳转关系及核心模块。

步骤 ii. 用户流程自动化

将 PRD 直接转化为可编辑的用户流程图 (User Flow)。

步骤 iii. 变量频率统计

统计竞品特定变量 (X/N)，得出概率级有效的结论证据。

生产级 PROMPT 代码库

prompt_logic_1.md 
【IA设计】：你是一名资深 UX设计师。根据PRD输出信息架构文档，包括：1.页面清单(每个页面一句话说明用途) 2.页面跳转关系 3.每个页面的核心模块。格式为 Markdown。
prompt_logic_2.md 
【竞品决策表】：基于竞品截图输出“设计决策依据表”，帮我完成半定量的变量频率统计(X/N)。从决策成本、信息密度、风险程度维度判断适配性，并生成专业汇报话术。
Visual Design 氛围编程专家 (Vibecoding)

UI 自动化与 VibeCoding

操作策略与核心步骤

“提取视觉基因并自动化覆盖系统可能存在的各种状态边缘。”

步骤 i. 提取设计变量

丢入参考图让AI分析色调、空间感、组件风格和情绪基调。

步骤 ii. UI 状态完备性

生成包含正常、报错、加载中等在内的 5 列 UI 状态清单。

生产级 PROMPT 代码库

prompt_logic_1.md 
【视觉提取】：你是一名资深 UI设计师。提取参考图中的视觉决策（色调、空间感、组件风格）并生成 Design Tokens (DESIGN_BRIEF.md + tokens.css)。
prompt_logic_2.md 
【状态走走清单】：生成UI状态覆盖清单（5列：状态、触发条件、UI组件、线上级文案、是否可恢复），覆盖正常路径、用户错误、系统错误、加载中、边缘决策状态。
Quality Audit 高级体验审计官

体验走查与审计

操作策略与核心步骤

“不仅限于表层评价，而是基于文档进行 system 重构与体验补齐建议。”

步骤 i. 无障碍性审计

检查对比度、触摸目标、键盘导航等 WCAG 合规性并提供修复 Prompt。

步骤 ii. 专家评审

自行拆解体验维度，从用户任务出发给出优化建议。

生产级 PROMPT 代码库

prompt_logic_1.md 
【无障碍性审计】：你是一名资深 UX/Accessibility 专家。请检查界面中的色彩对比度、触摸目标尺寸、文本层级、键盘导航。提供修复方案并生成迭代提示词。
prompt_logic_2.md 
【专家评审】：你是资深 UX 评审专家，按照产品文档，基于当前的设计方案，自行拆解体验维度，并给出优化建议。
